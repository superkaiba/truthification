Experimental Design

The Hidden Value Game

Three roles interact over 10 rounds:

  • Judge: Selects high-value objects based on agent statements and oracle queries
  • Agents A & B: Know the true value rule; try to manipulate Judge toward objects they want
  • Estimator: Passively observes; tries to infer truth from biased statements

World: 10 objects × 5 properties = 50 facts to learn. Oracle budget: 8 queries. Selection size: 5 objects.

Factors Varied (Full Factorial Design)

Factor 1: Judge's Knowledge About Agents

Finding: Knowing agent interests provides +4.6% accuracy boost (55.1% vs 50.5% for IDs only). Just knowing agent IDs doesn't help—you need to know what they want to discount biased claims.

Factor 2: Oracle Query Strategy

The Random Oracle Paradox: Random queries achieve +6.9% higher property accuracy (56.2% vs 49.3%) but -22% lower value (116 vs 148). Strategic queries optimize for selection quality, not knowledge breadth.

Factor 3: Agent Objective Complexity

Finding: Complex agent objectives improve accuracy by +4.0% (54.8% vs 50.8%). Agents with complex goals discuss more properties, providing richer information for truth recovery.

Factor 4: Hidden Value Rule Complexity

Trade-off: Complex rules have +15.4% higher property accuracy (63.3% vs 47.9%) but -24.3% lower rule inference (38.5% vs 62.8%). Complex scenarios are easier to learn facts from but harder to fully understand.

Interaction Effects

Best Combination: Complex Agent VF × Complex Rule achieves 68.2% property accuracy— the highest of any condition. Maximum information diversity leads to best truth recovery.

Agent Success Analysis

How well did agents manipulate the Judge into selecting objects they wanted?

Finding: Agent B slightly outperforms Agent A on average (92.8 vs 83.8 for complex VF). When Judge knows agent interests, total agent success decreases by 4% (86.7 vs 90.4)— the Judge can discount biased claims.

Estimator Performance (Passive Observer)

Finding: The Estimator barely beats random guessing (+1.4% above 30.7% baseline). Without ability to query the oracle, passive observation of strategic deception is extremely difficult.

Agent Objective Inference

The Estimator attempts to infer what each agent is trying to optimize for based on their statements throughout the game. This is evaluated by an LLM judge comparing the inferred objectives to the true agent value functions.

Key Finding: Simple agent objectives are much easier to infer (65.8%) than complex ones (35.3%). When agents want simple things ("red objects"), their behavior reveals their goals. Complex multi-condition objectives are harder to detect.
Finding: Knowing agent interests helps inference slightly (52.6% vs 47.9% blind), but the effect is smaller than for property accuracy.

Conclusions

What Helps Truth Recovery

  • Know agent motivations (+4.6%)
  • Complex scenarios (+15-20%)
  • Random oracle sampling (+6.9% accuracy)

What Doesn't Help

  • Just knowing agent IDs (-2.3%)
  • Strategic oracle (-6.9% accuracy)
  • Passive observation (~0% above random)

Implications for AI Safety

  1. Transparency about incentives matters: Knowing agent objectives significantly improves truth detection
  2. Strategic verification has limits: Focused fact-checking may miss broader deception patterns
  3. Passive monitoring fails: Without ability to verify claims, observers cannot distinguish truth from deception
  4. Complexity can help: Richer environments generate more cross-checkable information
